# 書籍摘要 — 執行計畫

## 背景
資料夾內共 **235 本電子書**（158 epub + 77 pdf），主題涵蓋兩性關係、心理學、搭訕理論等。目標：為每本書產出一份「精華摘要」markdown，濃縮成可快速查閱的筆記。由於數量龐大，不可能一次完成，採**分批＋持久化進度追蹤**的方式執行，可跨多個對話 session 繼續。

## 檔案結構
```
summaries/
├── PLAN.md              ← 本檔案（執行計畫）
├── PROGRESS.md           ← 進度追蹤總表（235本全清單＋checkbox＋分類，每批次結束後更新）
├── index.md              ← （後續建立）已完成摘要的一覽索引，含一句話簡介＋連結
├── {分類資料夾}/
│   └── {書名}.md          ← 個別摘要檔案
```

## 執行流程（每一批次）
1. 從 `PROGRESS.md` 挑選下一批 **8~10 本**（同分類優先，方便主題聚焦）
2. **平行委派**給多個 subagent（Task tool, general agent），每個負責一本書：
   - 來源檔案路徑
   - 目標輸出路徑（`summaries/{分類}/{書名}.md`）
   - 摘要模板（見下）
   - 明確要求：**必須用繁體中文撰寫**
3. 全部完成後，**強制跑 OpenCC `s2twp` 簡轉繁**做二次校正（不依賴 subagent 是否遵照指示，作為品質保險）
4. 更新 `PROGRESS.md`：勾選已完成項目、標註輸出路徑
5. 更新 `index.md`：加入本批次書籍的一句話摘要
6. 向你回報本批進度（完成幾本／剩餘幾本），繼續下一批

## 爭議內容處理原則（2026-07-09 確立）
部分書籍內容涉及物化女性、欺騙操控、忽視對方意願（同意）等高倫理爭議話術。**統一原則：仍完整摘要（含具體技巧），但以「批判性反例」角度呈現**：
- 標籤加註 `#反例警示`
- 摘要開頭加入內容警示區塊，說明本摘要供認識/辨識用途，非操作建議
- 「精華技巧」章節改為「書中技巧（批判角度標註）」，每項技巧後方加 ⚠️批判 小段，指出操控性質/問題所在
- 適讀情境章節補充「如何辨識/自我保護」的反向建議
不需要每次都詢問使用者，後續批次遇到類似內容直接套用此原則處理。
